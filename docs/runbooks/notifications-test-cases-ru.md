# Тест-кейсы уведомлений (EP-007)

Ручные тест-кейсы для EP-007: Notifications & Realtime.

## Предварительные требования

- [ ] Backend запущен на `http://localhost:8080`
- [ ] Веб-клиент запущен на `http://localhost:5173`
- [ ] Keycloak запущен и настроен
- [ ] Минимум 2 тестовых пользователя в одном домохозяйстве (alice, bob)
- [ ] PostgreSQL с миграциями V015-V017

## ST-601: SSE Authentication

### TC-001: SSE-сессия через HttpOnly Cookie

**Цель:** Проверить, что SSE-соединение аутентифицируется через HttpOnly cookie.

#### Предусловия
- Пользователь вошёл в систему
- Выбрано домохозяйство

#### Шаги

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Войти как alice | На странице домохозяйства |
| 2 | Открыть DevTools → Network | Мониторинг запросов |
| 3 | Наблюдать запрос к `/auth/session` | POST запрос с JWT в теле |
| 4 | Проверить Response Headers | `Set-Cookie: hometusk_session=...; HttpOnly; SameSite=Strict; Path=/` |
| 5 | Наблюдать SSE-соединение | GET `/households/{id}/notifications/stream` |
| 6 | Проверить Request Headers SSE | Cookie отправляется автоматически (без Authorization header) |

#### Верификация
- [ ] Cookie установлен с флагом HttpOnly
- [ ] SSE-соединение успешно установлено (статус 200, тип `text/event-stream`)
- [ ] В консоли нет ошибок CORS

### TC-004: Realtime уведомление о новой задаче

**Цель:** Проверить, что уведомление приходит в реальном времени при создании задачи.

- alice и bob в одном домохозяйстве

| 4 | alice: нажать на колокольчик | Выпадающий список с новым уведомлением |

#### Верификация
- [ ] Уведомление пришло в течение 1-2 секунд

- [ ] Уведомление содержит корректную информацию

## ST-604: Degraded Fallback

### TC-008: Переключение на polling при недоступности SSE

**Цель:** Проверить fallback на polling при недоступности SSE.

#### Шаги

| Шаг | Действие | Ожидаемый результат |
|-----|----------|---------------------|
| 1 | Войти и выбрать домохозяйство | SSE-соединение установлено |
| 2 | DevTools → Network → Block request URL `*/stream` | SSE заблокирован |
| 3 | Обновить страницу | SSE не может подключиться |
| 4 | Ждать ~10-15 секунд (5 ретраев) | Переключение на polling |
| 5 | Проверить header | Индикатор degraded mode (иконка облака с крестом) |
| 6 | Наблюдать Network | GET `/notifications?since=...` каждые 30 секунд |

#### Верификация
- [ ] После 5 неудачных попыток SSE переключается на polling
- [ ] Индикатор degraded mode виден
- [ ] Polling запросы отправляются каждые 30 секунд
