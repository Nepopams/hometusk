Owner: Planning/Architecture (Claude Code)
Epic: EP-009
Sprint: S08

## Goal
Повысить “доходимость” бытовых задач: сделать выполнение видимым и вознаграждаемым через очки/достижения/серии, не превращая продукт в детский сад и не ломая серьёзный командный UX.

## Why now
Без мотивации и обратной связи домашние задачи проигрывают привычке “потом”. Геймификация — это не про “веселье”, а про петлю подкрепления: сделал → получил признание/прогресс → продолжаешь.

## In Scope
- **Points (баллы) за задачи:**
  - базовая цена задачи (по типу/зоне/ручной настройке)
  - начисление при `completed`
  - отмена/коррекция (admin/manual) с логом причины
- **Badges / Achievements (минимум):**
  - “10 задач”, “7 дней подряд”, “герой кухни” (по зоне)
  - вычисление по событиям TaskActivity/Task completion
- **Streaks (серии):**
  - серия “день с выполненной задачей”
  - правила обнуления (простые, явно документированные)
- **UI:**
  - профиль участника: очки, серия, бейджи
  - недельный мини-лидерборд (без токсичности: “прогресс”, а не “унижение”)
- **Анти-чит и честность (база):**
  - нельзя набивать баллы массовым созданием микрозадач без ограничений (простые лимиты/правила)

## Out of Scope
- Денежные награды/кошельки/покупки
- Сложные квесты/сюжет/коллекции
- Детские режимы и “родительский контроль” как отдельный продукт
- Любая ИИ-персонализация

## Deliverables
- Backend:
  - таблицы/сущности: PointsLedger (начисления), Badge (каталог), UserBadges, StreakState
  - сервисы расчёта: PointsService, BadgeService, StreakService
  - endpoints: получить прогресс пользователя/household leaderboard
- Web:
  - вкладка/страница “Progress”
  - карточка участника + leaderboard
- Docs:
  - правила начисления, сброса серий, анти-чит ограничения

## Exit Criteria
1) При завершении задачи начисляются баллы и видны в профиле.
2) Есть минимум 5 достижений, которые автоматически открываются.
3) Серия считается корректно и предсказуемо.
4) Есть простой leaderboard за неделю.
5) Любые ручные корректировки логируются (кто/когда/почему).

## Success Metrics
- +20% выполненных задач/неделя на household (по сравнению с базовой неделей).
- ≥ 50% активных пользователей открыли хотя бы 1 бейдж за 7 дней.
- Низкий “уровень токсичности”: минимум конфликтов из-за лидерборда (качественный фидбек).

## Dependencies
- Нотификации (желательно): “получен бейдж”, “серия продлена”
- Корректный TaskActivity / audit trail

## Risks
- Токсичный leaderboard → показывать прогресс + опционально скрывать рейтинг.
- “Сломанная мотивация” из-за странных правил → правила должны быть простыми и документированными.
- Рост доменной сложности → держим минимальный набор механик в MVP.

## Candidate Stories
- ST-4xx: Points ledger + начисление при completion
- ST-4xx: 5 badges + автоматическое присвоение
- ST-4xx: Streak rules + расчёт
- ST-4xx: Progress endpoints + web UI
- ST-4xx: Admin/manual adjust + audit log
