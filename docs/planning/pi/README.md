# Program Increment (PI) Planning

This directory contains PI-level planning artifacts (if project follows SAFe/PI cadence).

## Structure

Each PI should have:
- `pi-{number}-objectives.md` — PI objectives and success metrics
- `pi-{number}-capacity.md` — Team capacity and velocity
- `pi-{number}-risks.md` — Identified risks and mitigations
- `pi-{number}-retrospective.md` — PI retrospective (post-PI)

## Example

```
pi/
  pi-1-objectives.md
  pi-1-capacity.md
  pi-1-risks.md
  pi-1-retrospective.md
```

## When to Use

Use PI planning if:
- Team follows SAFe/LeSS/Scrum@Scale
- Planning horizon is 8-12 weeks
- Multiple teams coordinate work

**Skip PI planning if**:
- Team is < 5 people
- Work is short-lived (< 4 weeks)
- Agile Kanban/continuous flow preferred
